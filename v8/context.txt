Why this occurred:
Chrome's on-device Prompt API (promptStreaming) streams responses differently depending on the browser version.

In some versions, the API streams accumulated text (the full sentence up to that point).
In other versions, the API streams deltas (individual new tokens/words as they are generated).
Our previous implementation assumed an accumulated stream. Because your Chrome version streams deltas, setting fullResponse = chunk ended up overwriting the bubble contents on every token, causing it to display only one word at a time and show only the final punctuation mark when the stream completed. Additionally, any minor warning thrown at the very end of the stream in the catch block would wipe out the generated text.

How I Fixed It:
Adaptive Stream Detection (v8/sidepanel.js): Modified the streaming loop to support both delta and accumulated chunks dynamically. It checks if the incoming chunk begins with the existing response. If it does, it updates the text (accumulated); if not, it appends the token (delta):
javascript


if (fullResponse && chunk.startsWith(fullResponse)) {
  fullResponse = chunk; // Accumulated chunk
} else {
  fullResponse += chunk; // Delta chunk
}
Resilient Error Boundary: Updated the stream's catch block. If the local model throws a non-critical error at the end of streaming, the code now appends the warning as a minor italicized alert instead of wiping out your generated content.
